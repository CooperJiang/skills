# recall — Claude 的记忆 / 上下文管家

## 背景痛点
用户走中转站,老会话的 session 绑上游账号,账号过期后那段对话**无法用 `claude -r` / `Ctrl+R` 恢复**(中转侧拒绝),只能**新开对话**。所以无法靠 resume 灌全量历史,得从本地 transcript(`~/.claude/projects/<编码项目路径>/*.jsonl`)里把上下文捞回来。本 skill 专门服务这件事,并顺带管理会话与持久记忆。

脚本在 `~/.claude/skills/recall/recall.py`,只用标准库,秒级(33MB 约 0.15s)。
**分工铁律:脚本只做笨活(解析/抽取/搬移文件);总结、判断什么该沉淀、写 memory 文件,都由你(模型)做。**

## 能力一:找回并接手一段对话(最常用)
两步,别跳:

**① 列出会话让用户挑**
```
python3 ~/.claude/skills/recall/recall.py list            # 当前项目
python3 ~/.claude/skills/recall/recall.py list --all      # 跨所有项目
python3 ~/.claude/skills/recall/recall.py list --archived # 连归档区一起列
python3 ~/.claude/skills/recall/recall.py list --json     # 结构化(给菜单用)
```
`✎`=用户 `/rename` 改过名(别名取自 `~/.claude/history.jsonl` 最新 `/rename`);没改名的用首条用户消息当标题。**把列表原样给用户**。

**用可导航菜单让用户选(而不是只让他打字)**:用 `AskUserQuestion` 弹**两问一屏**——
  - 问1「接手哪一段?」:选项 = 各会话(label=别名,description=`时间·分支·大小`)。**跳过当前会话**(它在列表里通常是第1条,是 recall 会话本身)。选项上限 4 个:会话 >4 段时,这一问改为让用户直接报序号,别硬塞。
  - 问2「读取细致程度?」:选项 = **简读 / 标准 / 详读 / 超详读**(见下)。这问永远能用菜单。
这样用户就能像 `claude -r` 一样导航选择(方向键/点击),选完对话接着选深度。**注意**:skill 无法唤起 `claude -r` 那个原生全屏方向键 TUI(它在 CLI 二进制里),`AskUserQuestion` 的选择卡是最接近的等效物。用户当然也可以直接打字「2 详读」。**别替用户选对话**(“最近一段”未必是他要的)。

**② 抽取选中会话 → 你合成简报**
```
python3 ~/.claude/skills/recall/recall.py show <序号|shortid|id> --depth <简读|标准|详读|超详读>
python3 ~/.claude/skills/recall/recall.py show 4 6 --depth 详读   # 一次拉多段合并
```
**读取深度 ≠ 读几轮尾巴**,而是「读哪些结构化信号」。一段对话的要害散落全程,尾巴只是其一。核心信号是**全局记忆**:`isCompactSummary`——Claude 上下文满时自动压缩出的整段梗概(每段可上万字,是这段对话的“数字记忆”)。深度就是控制取多少层:

| 深度 | --depth | 全局记忆(压缩摘要) | 用户意图主线 | 尾轮 | 文件 |
|---|---|---|---|---|---|
| 简读 | quick | 最近1段(截断3000字) | — | 6 | 12 |
| 标准 | normal(默认) | **全部** + 阶段小结 | — | 12 | 25 |
| 详读 | deep | 全部 | 最近40条 | 24 | 50 |
| 超详读 | full | 全部 | 全部 | 50 | 全部 |

也可 `--turns N` / `--files N`(0=全部)手动覆盖。`show` 输出(按深度分层):**🧠 全局记忆(压缩摘要)** → 阶段小结 → 改动文件 → 最后一次 TODO → **用户意图主线** → 结尾 N 轮原文。注:只有长到触发过压缩的会话才有全局记忆;短会话没有,靠尾轮+意图主线即全部。读完后**你**产出简报,粒度=**任务+进度+待办**:① 在做什么 ② 进度(改了哪些关键文件/做了哪些决定)③ 停在哪 / 下一步 TODO ④ 关联 `memory/` 里的铁律。要能让你直接接着写代码。

**沉淀持久全局记忆(每段对话一份)**:`show` 输出末尾会给出 digest 路径 `~/.claude/recall-digests/<sid>.md`。你合成完接手简报后,**把简报写进这个文件**——下次 `show` 会在最前面直接秀出这份「💾 已有持久全局记忆」,秒读、不必重扫 34MB。这就实现了“每段对话一个持久数字梗概”。

## 能力二:跨对话工作(合并多段上下文)
`show` 传**多个 id** 即可把几段会话的材料一起拉进当前对话,你合并后就能跨会话干活(例:把 A 的免费分组线 + B 的 relay 线合起来)。这是**上下文层**的合并,即时、不落盘。

## 能力三:沉淀进持久记忆库(跨会话长期可用)
当某段对话里有**值得所有未来会话都记住**的事实(架构决定、运维铁律、踩坑),把它蒸馏进持久记忆库:
```
python3 ~/.claude/skills/recall/recall.py memory   # 打印记忆库目录 + 现有文件 + MEMORY.md 索引
```
拿到目录后,**按记忆规范**(见系统提示里的 Memory 段:一文件一事实、带 frontmatter、`type` 分类、正文用 [[name]] 互链)在 `memory/` 下新建/更新 `*.md`,并在 `MEMORY.md` 加一行指针。写什么、要不要写由你判断;先查有没有已覆盖的文件,避免重复。**这才是真正的“记忆合并”——从一次性对话变成持久跨会话记忆。**

## 能力四:给 claude -r 瘦身(软删=归档 / 硬删=永久)
用户接手完老会话后想清理,避免 `claude -r` 越堆越多。**删除入口 = 用户对我说「归档 N」/「删掉 N」**(list 输出底部已提示)。
```
python3 ~/.claude/skills/recall/recall.py prune <id> [<id>...]          # 默认=软删(归档,可恢复)
python3 ~/.claude/skills/recall/recall.py restore <id> [<id>...]        # 从归档拉回
python3 ~/.claude/skills/recall/recall.py prune <id> --delete --force   # 硬删(永久,需 --force)
```
- **默认软删=归档**(移到 `~/.claude/recall-archive/`):立刻从 `claude -r` 消失,但原文还在、`recall show` 仍能读、`restore` 可拉回。**目前一律用这个。**
- **硬删受双重保护**:`--delete` 单独用会被脚本拒绝并提示,必须再加 `--force` 才真删。
- 护栏:脚本**绝不动当前会话**;归档区文件不会被重复归档。
- **你必须遵守的流程**:
  1. prune 前先 `list`,把「将要处理哪些(序号+别名+大小)」列给用户**确认**。
  2. **默认只软删(归档)**。除非用户**明确说“永久删除/硬删除/彻底删掉”**,否则绝不加 `--delete`。
  3. 真要硬删:先**说明“不可逆、无法 restore”并二次确认**,得到用户再次明确同意后才用 `--delete --force`,逐条报给用户核对。宁可归档。

## 其它
- 纯本地读写,和中转账号是否过期无关——这正是它能绕开 resume 失效的原因。
- `list --json` 输出结构化。`show` 的 key 支持 序号/短id/完整id;归档区的会话也能被 resolve 到。
- 编码规则:项目绝对路径里的 `/` 和 `.` 都换成 `-`,即 `~/.claude/projects/` 下的目录名。
